Basic Idea
  • Function-based programs are organized primarily around the func tions (methods, etc.) that do things. Data structures (objects) are considered seperate.
  • Object-based programs are organized around the types of obje ts that are used to represent data; methods are grouped by the type of object.
Philosophy
  • Idea (from the 1970s and before): An abstract data type is
    • A set of possible values (a domain) plus
    • A set of operations on those values (or their containers)
  • In IntList, the domain was a set of pairs: (head and tail), where head is an int and tail is a pointer to an IntLIst
  • The IntList operations consisted only of assigning to and accessing the two fields (head and tail)
  • In general, we prefer a purely procedural interface, where the functions (methods) do everything - no outside access to the internal representation (i.e. instance variables).
  • That way, implementor of a class and its methods has complete control over behavior of instances.
  • In Java, the preferred way to write the "operations of a type" is as instance methods 

The Account class in Python
class Account:
    balance = 0
    def __init__(self, balance0):
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        return self.balance

    def withdraw(self, amount):
        if self.balance < amount:
            raise ValueError \ ("Insufficient Funds")
        else:
            self.balance -= amount
        return self.balance

The account class in Java
public class Account{
    public int balance;

    public Account(int balance0){    <-- Account Constructor
        this.balance = balance0;
    }

    public int deposit(int amount){
        balance += amount;
        return balance;
    }

    public int withdraw(int amount){
        if (balance < amount)
            throw new IllegalStateException("Insufficient Funds");
        else
            balance -= amount;
        return balance;
        }
}

The pieces
  • Class Declaration defines a new type of object, i.e., new type of structured containier
  • Instance Variables such as balance are the simple containers within these objects (fields or components)
  • Instance methods, such as deposit and withdraw are like ordinary (static) methods that take an extra parameter (called this)
  • The new operator creates (instantiates) new objects, and initializes them using constructors
  • Constructors such as the method-like declaration of Account are special methods that are used only to initialize new instances. They take their arguments from the new expression. 
  • Method selection picks methods to call
    • For example: myAccount.deposit(100) tells us to call the method named deposit that is defined for the objected pointed to by myAccount

Static - variable that is is shared by ALL instances of the object (NOT an instance variable) 'class-wide'
Instance - Instance methods behave like static methods with an extra parameter

*You CANNOT refer to instance variables directly in a static method*
    You CAN access an instance variable in a static method by referencing the specific instance (typically passed in as a parameter)
Constructors
  • A constructor is a special kind of instance method that is called by the new operator right after it creates a new object.
  • ALL classes have constructors. If any constructor is not explicitly defined, the default constructor is provided. (Does Nothing)
  • It is possible to have multiple overloaded constructors, each constructor can use the others
Example of Overloaded Constructors
public class IntList {
    public IntList(int head, IntList tail) {
        this.head = head; this.tail = tail;
    }
    public IntList(int head) {
        this(head, null);     //<-- Calls first constructor
    }
}
